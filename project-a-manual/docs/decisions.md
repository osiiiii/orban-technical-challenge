Kept it simple like the brief said. Tags as JSON not a join table, one shared api key not accounts, tables made on startup not migrations, sqlalchemy so postgres is a one-liner later.Schemas split into create/update/out so clients can't touch id or timestamps, errors all one flat {detail} string. Also note that the keys never hits the browser, the frontend calls a proxy that adds it server-side.I skipped pagination, rate limiting and per-user stuff, none of it needed here.
