Notes with title/body/tags/timestamps..CRUD plus search by keyword or tag, simple key auth, sqlite, with decent looking errors. the frontend that lists/filters/creates/edits and shows failures, a few tests and docs. "Simple" was in the brief twice so the job was mostly not overbuilding it. There is a thin router with logic in separate files, one shared form for create and edit, an api client that throws and the pages catch.The tests were focused on what could actually break, auth, blank title, the full crud run, search, each on a fresh in-memory db. One gotcha, /notes/search has to come before /notes/{id} or fastapi eats it.
